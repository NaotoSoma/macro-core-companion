# `GECoordinationWidget` 実装仕様書

## 1. 目的

第1回「一般均衡とは何か」の講義ページに埋め込む対話型部品として、次の3点を**視覚的かつ操作的に**伝える。

1. 家計・企業・政府の意思決定は、それぞれの市場では1本の線として表現できる。
2. 1つの市場における2本の線の交点が部分均衡である。
3. 3つの市場の交点を同時に考えた価格と量の組が一般均衡である。

この部品は、数学的な厳密さよりも、**一般均衡の見取り図を直感的に掴ませること**を優先する。

---

## 2. この widget（対話型部品）が置かれる文脈

対象ページは `src/content/docs/lectures/01-general-equilibrium/index.mdx` であり、本文ではすでに以下を説明している。

- 第1回は数式を使わず、一般均衡の見取り図をつかむ回である。
- 主体は家計・企業・政府・中央銀行だが、最低限同時に確認する市場は財市場・労働市場・資産市場の3つである。
- 第1回の主眼は「モデルを閉じる」とは何かを理解することにある。

したがって、本 widget の責務は**一般均衡の概念的可視化**であり、家計や企業の最適化問題の導出や、中央銀行の政策ルールの導入までは行わない。

## 4. スコープ

### 4.1 今回の実装に含めるもの

- 財市場・労働市場・資産市場の3つの価格・数量図
- 家計・企業・政府の表示切替
- 「曲線」「部分均衡」「一般均衡」の3モード
- 各市場における2本の線と交点の表示・非表示
- 下部の要約欄での均衡結果の文章化
- 最低限のアクセシビリティ対応

### 4.2 今回の実装に含めないもの

- 厳密なミクロ的導出
- 中央銀行を1本の曲線として描くこと
- 実データとの連動
- 複雑なアニメーション
- 数値的に厳密な一般均衡計算
- 外部状態管理ライブラリ
- 外部チャートライブラリ

## 5. 学習上の設計方針

この部品は「数値を読む部品」ではなく、「概念の違いを理解する部品」である。したがって、以下の順序で理解が進むように設計する。

1. **線を見る**  
   誰の行動がどの線に対応するかを見る。
2. **交点を見る**  
   1市場だけを見た交点が部分均衡だと理解する。
3. **同時性を見る**  
   3市場を同時に見た交点の組が一般均衡だと理解する。

## 7. 表示する線

### 7.1 財市場
- 家計需要曲線
- 企業供給曲線
- 政府需要の有無は、家計需要曲線を右方にシフトさせる形で表現する

### 7.2 労働市場
- 家計の労働供給曲線
- 企業の労働需要曲線
- 政府の線は描かない

### 7.3 資産市場
- 家計の貯蓄供給曲線
- 企業の資金需要曲線
- 政府を表示した場合、資金需要曲線を右方にシフトさせる

## 8. 色と意味付け

色分けは**需要・供給ベースではなく主体ベース**にする。

- 家計由来の線: 青系
- 企業由来の線: 橙系または赤系
- 政府の寄与: 緑系の補助表示
- 交点: 黒または濃い強調色
- 非アクティブ市場: 低彩度グレー寄り

理由は、第1回で最も重要なのが「誰の意思決定がどの線か」を理解することだからである。

## 9. モード仕様

widget は次の3モードを持つ。

### 9.1 モードA: 曲線

#### 目的
「主体の意思決定が線になる」ことを理解させる。

#### 挙動
- 各市場に線だけを表示する。
- 交点は表示しない。
- 需要・供給のどちらか一方しか表示されない場合も、そのまま線だけを見せる。
- 下段の要約欄は「いまは線だけを見ています」と説明する。

#### 学習メッセージ
- 家計は需要・供給・貯蓄など、複数市場で線を持つ。
- 企業も供給・需要・資金需要として複数市場に現れる。

### 9.2 モードB: 部分均衡

#### 目的
1市場だけを見た交点を部分均衡と理解させる。

#### 挙動
- 1つの市場をフォーカス対象として選べる。
- フォーカス市場は通常濃度で表示する。
- その他2市場は薄く表示する。
- フォーカス市場で需要曲線と供給曲線の両方が存在する場合のみ、交点を表示する。
- 交点から縦軸・横軸へ補助線を引く。
- 軸ラベルの横に「価格」「量」またはそれに準ずるラベルを表示する。
- 下段の要約欄には「この市場だけを見た交点が部分均衡」と表示する。

#### 学習メッセージ
- 財市場だけ、労働市場だけ、資産市場だけを見ることもできる。
- しかし、それだけでは他市場との整合性はまだ確認していない。

### 9.3 モードC: 一般均衡

#### 目的
3市場を同時に見た交点の組が一般均衡であると理解させる。

#### 挙動
- 3市場すべてを通常濃度で表示する。
- 各市場で2本の線が見えている場合、その市場の交点を表示する。
- 3市場すべてで交点が描けるときのみ、「一般均衡が成立」と明示する。
- 1つでも必要な線が欠けている場合は、「一般均衡を確認するには、各市場で両側の線が必要です」と表示する。
- 下段の要約欄には、3市場それぞれの価格と量の組を並べて表示する。

#### 学習メッセージ
- 一般均衡は「1つの巨大な交点」ではなく、複数市場の交点の組である。
- 重要なのは、これらが**同時に**整合していること。

- ラベル表示 OFF の場合、線ラベルと交点ラベルを消す。

## 12. 数学仕様（厳密性より見やすさ優先）

この widget は厳密な均衡計算器ではない。線は**教育用の簡約関数**として固定パラメータで描画する。

### 12.1 座標空間

各市場は共通して正方形の SVG 座標空間を使う。

- 内部座標: `x, y ∈ [0, 100]`
- 描画余白: 上 10, 右 8, 下 14, 左 14 程度

### 12.2 線の形

初版ではすべて直線でよい。

#### 財市場
- 需要: 右下がり直線
- 供給: 右上がり直線
- 政府 ON 時: 需要曲線の x 切片を右にずらす

#### 労働市場
- 労働供給: 右上がり直線
- 労働需要: 右下がり直線

#### 資産市場
- 貯蓄供給: 右上がり直線
- 資金需要: 右下がり直線
- 政府 ON 時: 資金需要を右にずらす

### 12.3 例示用パラメータ

実装は以下のような係数で十分。

```ts
// 例: y = a + b*x
goodsDemandBase    = { a: 78, b: -0.60 }
goodsSupplyBase    = { a: 18, b:  0.55 }
laborSupplyBase    = { a: 16, b:  0.62 }
laborDemandBase    = { a: 82, b: -0.58 }
assetsSupplyBase   = { a: 22, b:  0.50 }
assetsDemandBase   = { a: 76, b: -0.52 }
```

政府 ON 時のシフト量の目安:

```ts
goodsDemandShiftX = +10
assetsDemandShiftX = +8
```

### 12.4 交点計算

2直線の交点を計算し、SVG 座標へ変換する。

```ts
function intersect(line1, line2) {
  // line: y = a + b*x
  const x = (line2.a - line1.a) / (line1.b - line2.b)
  const y = line1.a + line1.b * x
  return { x, y }
}
```

### 12.5 交点の表示条件

- 線が2本とも表示されていること
- 交点が描画領域内にあること
- `showIntersections === true` であること

## 15. UI 文言仕様

### 15.1 ヘッダー

- タイトル: `部分均衡と一般均衡を見比べる`
- 補足: `線 → 交点 → 同時整合性、の順に見てください。`

### 15.2 モード別説明

#### 曲線モード
`いまは、家計や企業の意思決定が各市場でどのような線として現れるかを見ています。`

#### 部分均衡モード
`いまは 1 つの市場だけに注目しています。この市場だけを見た交点が部分均衡です。`

#### 一般均衡モード
`いまは 3 つの市場を同時に見ています。各市場の交点の組が一般均衡です。`

### 15.3 要約欄の文言

#### 曲線モード
`まだ交点には注目していません。どの主体がどの市場に線を持つかを確認してください。`

#### 部分均衡モード
`この市場だけを見た価格と量の組が部分均衡です。他の市場との整合性はまだ確認していません。`

#### 一般均衡モード（成立時）
`3つの市場で同時に整合する価格と量の組を一般均衡と呼びます。`

#### 一般均衡モード（未成立時）
`一般均衡を確認するには、各市場で両側の線が見えている必要があります。`

## 19. パフォーマンス方針

- 外部チャートライブラリは使わない
- SVG を直接 React で描く
- 状態は widget 内のローカル state のみで持つ
- 再計算は単純な線形交点だけなので `useMemo` は必要最低限でよい

この程度の部品では、保守性の観点からも素の SVG 実装を優先する。

## 21. 擬似コード

```tsx
export default function GECoordinationWidget() {
  const [mode, setMode] = useState<WidgetMode>('curves');
  const [focusMarket, setFocusMarket] = useState<MarketKey>('goods');
  const [visibility, setVisibility] = useState<VisibilityState>({
    household: true,
    firm: true,
    government: false,
  });
  const [display, setDisplay] = useState<DisplayOptions>({
    showIntersections: true,
    showGuides: true,
    showLabels: true,
  });

  const markets = buildMarkets({ visibility });
  const equilibria = computeEquilibria(markets);

  return (
    <section className="ge-widget not-content">
      <Header />
      <ControlPanel
        mode={mode}
        setMode={setMode}
        focusMarket={focusMarket}
        setFocusMarket={setFocusMarket}
        visibility={visibility}
        setVisibility={setVisibility}
        display={display}
        setDisplay={setDisplay}
      />

      <div className="ge-market-grid">
        {(['goods', 'labor', 'assets'] as MarketKey[]).map((marketKey) => (
          <MarketPanel
            key={marketKey}
            marketKey={marketKey}
            mode={mode}
            focus={focusMarket}
            lines={markets[marketKey]}
            equilibrium={equilibria[marketKey]}
            dimmed={mode === 'partial' && focusMarket !== marketKey}
            display={display}
          />
        ))}
      </div>

      <EquilibriumSummary
        mode={mode}
        focusMarket={focusMarket}
        equilibria={equilibria}
        visibility={visibility}
      />
    </section>
  );
}
```

- 数値は表示しない

## 24. 受け入れ基準

以下をすべて満たしたら初版完成とする。

1. ページ読み込み時に widget が崩れず表示される。
2. 3市場が可視化される。
3. 家計・企業・政府の ON/OFF で線が切り替わる。
4. `曲線 / 部分均衡 / 一般均衡` の切替が動作する。
5. 部分均衡モードでフォーカス市場の交点だけを見せられる。
6. 一般均衡モードで3市場の交点の組を要約欄に表示できる。
7. 政府 ON 時に財市場と資産市場で線のシフトが見える。
